# AI Doc Agent для Damumed Mock

Внешний AI-слой поверх медицинской системы в формате Chrome Extension Manifest V3. Проект сделан под кейс AIS Hack 3.0 HealthTech & RPA для демонстрации RPA-агента, который читает DOM, слушает врача, структурирует диктовку, заполняет формы, строит расписание и сохраняет данные только после подтверждения человека.

## Структура проекта

```text
ai-doc-agent/
├── manifest.json                 # Manifest V3, точка входа Chrome Extension
├── background.js                 # Service worker: orchestration, OpenAI proxy, state machine
├── content_script.js             # DOM RPA: скрейпинг страницы, поиск полей, заполнение форм
├── popup.html                    # Popup для настройки Render proxy и запуска demo mock
├── popup.js                      # Логика popup, сохранение настроек и проверка AI proxy
├── icons/                        # Иконки расширения
├── mock/                         # Mock-страницы Damumed
│   ├── patient-card.html         # Карточка пациента и первичный осмотр
│   ├── schedule.html             # Назначения / расписание
│   ├── services.html             # Журнал услуг и статусы выполнено
│   ├── epicrisis.html            # Выписной эпикриз
│   └── mock-style.css            # Дизайн mock-интерфейса в стиле Damumed
├── modules/
│   ├── context-engine.js         # Классификация речи и сбор контекста
│   ├── decision-engine.js        # Выбор инструмента: scribe, navigator, scheduler
│   ├── scheduler.js              # 9-дневный deterministic scheduler
│   ├── selectors.js              # Семантический DOM-поиск полей и кнопок
│   ├── safety-gate.js            # Human-in-the-loop safety gate перед сохранением
│   ├── step-machine.js           # Состояния приема
│   └── template-manager.js       # Хранение шаблонов врача
├── overlay/
│   ├── overlay.js                # Плавающая панель AI Doc Agent
│   ├── overlay.css               # UI overlay
│   └── overlay.html              # Резервная HTML-разметка overlay
├── prompts/                      # Промпты для будущего расширения модулей
├── tests/
│   └── scheduler-smoke.js        # Быстрая проверка 9-дневного расписания
└── proxy-server/                 # Render proxy backend для OpenAI API
```

## Как запустить расширение в Chrome

1. Открой Chrome.
2. Перейди в:

```text
chrome://extensions
```

3. Включи `Режим разработчика`.
4. Нажми `Загрузить распакованное расширение`.
5. Выбери папку проекта:

```text
ai-doc-agent
```

Важно: выбирать нужно именно папку, где лежит `manifest.json`.

6. Нажми на иконку расширения `AI Doc Agent`.
7. В popup укажи:

```text
Render proxy URL:
https://web-service-case.onrender.com

Client token:
AISHack

Model:
gpt-4o-mini
```

8. Нажми `Сохранить настройки`.
9. Нажми `Проверить AI proxy`.
10. Если все подключено, popup покажет:

```text
AI proxy работает.
Provider: render-proxy
Model: gpt-4o-mini
```

11. Нажми `Открыть demo mock`.

## Demo flow

1. На mock-странице нажать `Начать прием`.
2. Нажать `Включить микрофон`.
3. Нажать `Начать запись` или сказать:

```text
начни запись
```

4. Продиктовать пример:

```text
Иванов Артем Сергеевич, 8 лет. Ребенок жалуется на слабость в ногах, быструю утомляемость при ходьбе, снижение активности к вечеру. Проходит повторный курс реабилитации, ранее отмечалась положительная динамика после ЛФК и массажа. Состояние удовлетворительное, сознание ясное, мышечный тонус нижних конечностей повышен, координация движений снижена, походка неустойчивая. Рекомендуется ЛФК, массаж нижних конечностей, консультация психолога. Продолжить курс реабилитации, соблюдать режим нагрузок, выполнять домашние упражнения.
```

5. Сказать:

```text
разбери в поля
```

или нажать `Разобрать и подготовить preview`.

6. Проверить preview.
7. Сказать:

```text
подтверди
```

или нажать `Подтвердить и сохранить`.

8. Для расписания сказать:

```text
сформируй расписание
```

Или нажать кнопку `Сформировать расписание`.

После подтверждения агент переходит на страницу расписания и заполняет таблицу.

## Как показать, что расписание работает

### Через интерфейс

1. Открыть `mock/patient-card.html`.
2. Нажать `Начать прием`.
3. Вставить или продиктовать план с процедурами:

```text
Рекомендуется ЛФК, массаж нижних конечностей, консультация психолога.
```

4. Нажать `Разобрать и подготовить preview`.
5. Проверить, что в preview есть:

```text
procedures: ЛФК, массаж нижних конечностей, консультация психолога
```

6. Нажать `Сформировать расписание`.
7. В preview появится расписание на 9 рабочих дней.
8. Нажать `Подтвердить и сохранить`.
9. Агент откроет страницу расписания и заполнит таблицу.

### Через smoke-test

Из папки проекта:

```bash
node tests/scheduler-smoke.js
```

Ожидаемый результат:

```text
PASS Base rehab course
rows=21, days=9, unassigned=0

PASS Synonyms
rows=21, days=9, unassigned=0
```

## Что уже реализовано

- Chrome Extension Manifest V3.
- Плавающий overlay поверх страницы.
- Web Speech API для русской голосовой диктовки и команд.
- Persist-состояние overlay между переходами страниц.
- Скрейпинг контекста страницы: ФИО, возраст, диагноз, предыдущие записи.
- AI parsing через Render proxy и OpenAI `gpt-4o-mini`.
- Fallback heuristic parsing, если AI недоступен.
- Семантический поиск полей без жесткой привязки к id/class.
- Human-in-the-loop preview перед сохранением.
- Safety gate: агент не сохраняет форму без подтверждения врача.
- Голосовая навигация по вкладкам.
- Smart scheduling на 9 рабочих дней.
- Автодобавление строк расписания в DOM.
- Mock-интерфейс, визуально похожий на Damumed: боковая панель, зеленые акценты, медицинский glow UI.

## Как это внедрять в реальный Damumed

Проект не требует доступа к API Damumed. Он работает как внешний RPA-слой поверх DOM:

```text
Damumed page
↓
Content script читает DOM
↓
AI Coordinator понимает контекст
↓
Tool-модули выполняют действия
↓
Preview и подтверждение врача
↓
Content script заполняет реальные поля Damumed
```

Для реального внедрения нужно:

1. Собрать HTML-карты реальных страниц Damumed:
   - карточка пациента;
   - первичный прием;
   - дневники;
   - назначения;
   - расписание;
   - эпикриз;
   - журнал услуг.

2. Расширить `modules/selectors.js`:
   - добавить реальные label/aria/placeholder признаки;
   - усилить поиск вкладок по тексту;
   - добавить fallback-поиск по соседним DOM-блокам.

3. Настроить `content_script.js` под реальные сценарии:
   - сохранение формы;
   - переходы между вкладками;
   - обработка модальных окон;
   - подтверждения и уведомления системы.

4. Оставить OpenAI API key только на backend proxy:
   - ключ нельзя хранить внутри Chrome Extension;
   - extension должен ходить только в Render/сервер;
   - сервер уже передает запросы в OpenAI.

5. Добавить аудит:
   - лог действий агента;
   - кто подтвердил форму;
   - время заполнения;
   - какие поля были изменены.

## Будущее развитие

### 1. Более сильный медицинский scribe

- Разделение речи на жалобы, анамнез, объективный статус, план, рекомендации.
- Поддержка русского и казахского.
- Медицинская терминология без бытовой речи.
- Строгие JSON schema ответы от модели.

### 2. PDF и внешние документы

- Подключить PDF.js на стороне extension.
- Извлекать текст анализов, выписок, справок.
- Передавать текст в proxy и добавлять в контекст пациента.

### 3. Шаблоны врача

- Хранить старые шаблоны в `chrome.storage.local`.
- Делать подбор шаблона по диагнозу.
- Генерировать эпикриз в стиле конкретного специалиста.

### 4. Реальное расписание

- Учитывать занятость специалистов.
- Добавить запрет тяжелых процедур подряд.
- Минимизировать пустые окна.
- Показывать плохое расписание и оптимизированное расписание для demo wow effect.

### 5. Тайм-менеджмент статусов

- Фиксировать начало и конец процедуры.
- Генерировать краткий дневник процедуры.
- Ставить `Выполнено` только после подтверждения.

### 6. Production hardening

- Добавить E2E-тесты на mock DOM.
- Добавить версионирование selectors.
- Добавить безопасный режим read-only.
- Добавить журнал ошибок и replay demo-сценариев.

## Безопасность

- Extension не хранит OpenAI API key.
- OpenAI key должен лежать только в backend env.
- Агент не нажимает `Сохранить` без human approval.
- Врач всегда видит preview перед записью данных в DOM.
- Для реального Damumed нужна отдельная проверка по требованиям клиники и локальному законодательству по медицинским данным.

## Backend proxy

Текущий Render proxy:

```text
https://web-service-case.onrender.com
```

Endpoint:

```text
POST /api/parse-dictation
```

Headers:

```text
x-client-token: AISHack
```

Backend хранит `OPENAI_API_KEY` в env и не отдает ключ клиенту.
